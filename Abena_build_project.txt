Below are 15 phased prompts you can give to any agentic IDE like Cursor, Claude Code, Windsurf, Replit Agent, etc. Each phase is written as a builder prompt, so you can copy and paste phase by phase.
I designed it so the builder can choose whether to use offline AI with Ollama/local models or API-based AI like OpenAI/Gemini/Groq.

Phase 1 Prompt: Project Setup and Architecture
You are building an AI-powered local business assistant for Ghanaian SMEs.

Project name: BizMate AI

Build a full-stack web application where business owners can upload their business information, products, services, prices, FAQs, and documents. Customers can then ask questions through a chatbot, and the AI answers using Retrieval-Augmented Generation.

The system must support both possible AI modes:
1. Offline/local AI mode using Ollama or local models.
2. API-based AI mode using OpenAI, Gemini, Groq, Claude, or another provider.

Do not hardcode one AI provider. Create an AI provider abstraction so the builder can choose later through environment variables.

Set up the project with this structure:

bizmate-ai/
  backend/
    app/
      auth/
      businesses/
      products/
      services/
      faqs/
      documents/
      chat/
      rag/
      ai_providers/
      database/
      utils/
      main.py
    requirements.txt
    .env.example

  frontend/
    app/
    components/
    lib/
    services/
    styles/
    package.json

  docs/
    README.md
    methodology.md

Recommended stack:
Frontend: Next.js with TypeScript and Tailwind CSS
Backend: FastAPI with Python
Database: PostgreSQL
Vector DB: ChromaDB or FAISS
Embeddings: Sentence Transformers or provider-based embeddings
LLM: configurable local or API-based provider

Tasks:
1. Initialize frontend and backend folders.
2. Set up FastAPI backend.
3. Set up Next.js frontend.
4. Configure Tailwind CSS.
5. Create .env.example files for frontend and backend.
6. Add basic README with project overview.
7. Add CORS configuration in backend.
8. Add health check route: GET /health.
9. Make sure frontend can call backend health route.
10. Keep the code modular and clean.

Important:
Do not build the AI logic yet.
Only create the foundation and project structure.

Phase 8 Prompt: RAG Indexing Pipeline
Continue building BizMate AI.

Now implement the RAG indexing pipeline.

The goal is to convert each business's products, services, FAQs, profile information, and uploaded documents into searchable vector data.

Use ChromaDB or FAISS as the vector database.
Choose the simpler and more stable option for this project.
Make the vector store modular so it can be changed later.

Tasks:
1. Create rag module.
2. Create text conversion functions for:
   - business profile
   - products
   - services
   - FAQs
   - documents

Example product text:
Product: HP EliteBook 840 G6.
Category: Laptop.
Price: GHS 4,200.
Availability: Available.
Quantity: 5.
Warranty: 3 months.
Description: Core i5, 8GB RAM, 256GB SSD.

Example FAQ text:
Question: Do you deliver?
Answer: Yes, we deliver within Kumasi. Delivery fee starts from GHS 15.

3. Create chunking function:
   - Product and service can be one chunk each.
   - FAQ can be one chunk each.
   - Documents should be split into chunks of 300–500 words.

4. Create metadata for each chunk:
   - business_id
   - source_type
   - source_id
   - title
   - created_at
   - updated_at

5. Generate embeddings using the selected embedding provider.
6. Store chunks and embeddings in vector database.
7. Implement function:
   index_business_data(business_id)

8. Implement function:
   delete_business_index(business_id)

9. Implement endpoint:
   POST /businesses/{business_id}/rag/reindex

10. When reindexing:
   - Delete old vectors for that business.
   - Rebuild all vectors from current data.

11. Add logs for indexing process.

Important:
The RAG index must separate data by business_id.
A customer chatting with one business must never retrieve another business's data.

Phase 9 Prompt: RAG Retrieval and AI Chat Engine
Continue building BizMate AI.

Now implement the customer AI chat engine using RAG.

Backend endpoint:

POST /chat/{business_id}

Request:
{
  "message": "Do you have HP laptops?",
  "customer_name": "optional",
  "customer_phone": "optional",
  "channel": "web"
}

Response:
{
  "answer": "...",
  "confidence_score": 0.87,
  "sources": [],
  "escalated": false
}

Chat flow:
1. Receive customer message.
2. Create or continue chat session.
3. Check if customer wants human support.
4. Convert question into embedding.
5. Retrieve top relevant chunks from vector database for the specific business_id.
6. Calculate confidence score based on retrieval similarity.
7. If confidence is low, return a safe fallback answer.
8. If confidence is acceptable, build a prompt with retrieved context.
9. Generate answer using selected LLM provider.
10. Save customer message and AI answer in chat history.
11. Return answer to customer.

Prompt rules:
The AI must:
- Act as a polite customer support assistant.
- Answer only using the provided business context.
- Never invent prices, products, services, discounts, or availability.
- If information is missing, say it does not have that information.
- Keep responses short and useful.
- Ask a helpful follow-up question when appropriate.
- Escalate to human if needed.

Confidence logic:
- If top score is below threshold, do not generate a confident answer.
- Use env variable:
  RAG_CONFIDENCE_THRESHOLD=0.50

Human handoff detection:
Detect messages like:
- human
- agent
- staff
- call me
- I want to talk to someone
- manager

If detected:
- create escalation record
- return a handoff response

Also handle sensitive categories:
For pharmacies, do not give medical diagnosis or prescriptions.
For legal/business-sensitive questions, avoid giving legal advice.

Important:
Make sure chat history is stored in the database.
Make sure retrieval only uses the selected business_id.
